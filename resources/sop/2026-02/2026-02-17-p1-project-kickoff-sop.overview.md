# L1 Overview - 项目启动与范围风险对齐

## When to use
- 新项目立项或重大版本开工
- 范围变更请求 >= 2 次

## Inputs
- Input 1: 项目目标与边界
- Input 2: 资源约束与里程碑

## Outputs
- Output 1: 项目启动包（范围、风险、依赖）
- Output 2: 启动评审结论与行动清单

## Minimal procedure
1) 收集项目目标、约束、干系人
2) 明确范围和非目标
3) 识别Top风险和缓解动作
4) 列出关键依赖与负责人
5) 召开启动评审并确认里程碑
6) 发布启动包并进入执行

## Quality gates
- Non-negotiables (legal/safety/security/data integrity) are explicitly checked.
- Objective is explicit and measurable.
- Outcome metric includes baseline and target delta.

## Invocation
`按SOP执行：项目启动与范围风险对齐 <输入>`
