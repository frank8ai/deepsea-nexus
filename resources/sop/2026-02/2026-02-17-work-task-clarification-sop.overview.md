# L1 Overview - 工作任务澄清与成功标准

## When to use
- a new task arrives without explicit KPI or acceptance criteria
- more than 2 clarification loops occurred for the same task

## Inputs
- Input 1: raw task request
- Input 2: constraints and deadline context

## Outputs
- Output 1: completed task-clarification record
- Output 2: approved success criteria and non-goals

## Minimal procedure
1) 收集任务原文与上下文
2) 提取目标、成功指标、截止时间
3) 补齐约束与非目标
4) 执行二轮复述确认
5) 生成任务澄清卡并冻结MVP边界
6) 将任务路由到计划SOP

## Quality gates
- Non-negotiables (legal/safety/security/data integrity) are explicitly checked.
- Objective is explicit and measurable.
- Outcome metric includes baseline and target delta.

## Invocation
`按SOP执行：工作任务澄清与成功标准 <输入>`
