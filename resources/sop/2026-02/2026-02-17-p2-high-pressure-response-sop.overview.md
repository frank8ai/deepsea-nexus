# L1 Overview - 高压事件处置与恢复

## When to use
- 出现高压事件信号或重大异常
- 事件影响扩散到多个关键链路

## Inputs
- Input 1: 事件信号和影响范围
- Input 2: 当前系统状态和沟通对象

## Outputs
- Output 1: 处置结论和沟通记录
- Output 2: 恢复验证报告和复盘规则

## Minimal procedure
1) 接收事件并快速分级
2) 执行止损动作
3) 对内外发布同步信息
4) 执行恢复动作并验证
5) 持续监控直到稳定
6) 24小时内复盘并回写规则

## Quality gates
- Non-negotiables (legal/safety/security/data integrity) are explicitly checked.
- Objective is explicit and measurable.
- Outcome metric includes baseline and target delta.

## Invocation
`按SOP执行：高压事件处置与恢复 <输入>`
